## 三、Vercel 部署步骤

### 3.1 准备工作

1. 注册 Vercel 账号（建议用团队公共 GitHub 账号）
2. 将代码推送到 GitHub 仓库

### 3.2 部署操作

```bash
# 1. 安装 Vercel CLI
npm i -g vercel

# 2. 进入项目目录
cd apps/pilot-h5

# 3. 登录 Vercel
vercel login

# 4. 首次部署（会自动检测 Vite 项目）
vercel

# 5. 确认配置：
#    - Framework: Vite
#    - Build Command: npm run build
#    - Output Directory: dist

# 6. 部署到生产环境
vercel --prod
```

### 3.3 环境变量配置

在 Vercel 控制台 → Settings → Environment Variables 中添加：

| 变量名 | 值 | 环境 |
|---|---|---|
| `VITE_SUPABASE_URL` | 你的 Supabase 项目 URL | Production |
| `VITE_SUPABASE_ANON_KEY` | 你的 Supabase anon key | Production |

### 3.4 自定义域名（可选）

1. 在 Vercel 控制台 → Settings → Domains 添加域名
2. 按提示在域名服务商添加 CNAME 记录
3. Vercel 自动配置 HTTPS 证书

## 五、自有服务器部署（Nginx）

如果使用自有服务器，Nginx 配置参考：

```nginx
server {
    listen 80;
    server_name pilot.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name pilot.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/pilot.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/pilot.yourdomain.com/privkey.pem;

    root /var/www/pilot-h5/dist;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml;
}
```

## 七、使用说明（发给飞手）

### 云深飞运 · 飞手端使用指南

#### 第一步：打开应用
- 收到链接后，点击打开
- 推荐添加到手机桌面：点击右上角 `⋯` → 添加到桌面

#### 第二步：登录
- 输入手机号
- 点击「获取验证码」
- 输入收到的验证码
- 点击「登录」

#### 第三步：查看订单
- 登录后进入「接单大厅」
- 可按状态筛选：待接单 / 已接单 / 执行中 / 已完成
- 点击订单卡片查看详情

#### 第四步：接单
- 在订单详情页点击「接受订单」
- 确认后订单状态变为「已接单」

#### 第五步：执行任务
- 点击「开始执行」进入任务执行页
- 按步骤操作：到达取货点 → 装货 → 飞行 → 送达
- 每步需确认并可上传照片

#### 第六步：完成任务
- 送达后点击「确认送达」
- 任务状态自动变为「已完成」

#### 常见问题
- **收不到验证码**：检查手机号是否正确，或联系管理员
- **页面打不开**：检查网络连接，或尝试刷新
- **定位不准**：可手动输入地址

## 八、Supabase 数据库初始化

部署前需确保 Supabase 数据库已创建必要的表。执行以下 SQL：

```sql
-- 订单表
CREATE TABLE IF NOT EXISTS orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_no TEXT UNIQUE NOT NULL,
  user_id UUID REFERENCES auth.users(id),
  pilot_id UUID REFERENCES auth.users(id),
  status TEXT DEFAULT 'pending',
  cargo_type TEXT NOT NULL,
  cargo_weight DECIMAL NOT NULL,
  cargo_desc TEXT,
  pickup_location TEXT NOT NULL,
  pickup_lat DECIMAL,
  pickup_lng DECIMAL,
  delivery_location TEXT NOT NULL,
  delivery_lat DECIMAL,
  delivery_lng DECIMAL,
  altitude DECIMAL,
  elevation_diff DECIMAL,
  lifting_method TEXT,
  price DECIMAL NOT NULL,
  remark TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 飞手表
CREATE TABLE IF NOT EXISTS pilots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) UNIQUE NOT NULL,
  name TEXT NOT NULL,
  phone TEXT NOT NULL,
  avatar TEXT,
  status TEXT DEFAULT 'offline',
  rating DECIMAL DEFAULT 5.0,
  total_orders INTEGER DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- RLS 策略
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;
ALTER TABLE pilots ENABLE ROW LEVEL SECURITY;

-- 所有登录用户可读订单
CREATE POLICY "Authenticated users can read orders"
  ON orders FOR SELECT
  TO authenticated
  USING (true);

-- 用户可创建订单
CREATE POLICY "Users can create orders"
  ON orders FOR INSERT
  TO authenticated
  WITH CHECK (auth.uid() = user_id);

-- 飞手可更新订单状态
CREATE POLICY "Pilots can update order status"
  ON orders FOR UPDATE
  TO authenticated
  USING (true)
  WITH CHECK (true);

-- 飞手可读自己的信息
CREATE POLICY "Pilots can read own info"
  ON pilots FOR SELECT
  TO authenticated
  USING (auth.uid() = user_id);
```
